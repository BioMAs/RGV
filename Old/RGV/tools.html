RGV genome browser (JBrowse)
details
Description

The implementation of JBrowse allows users to view and download datasets embedded into the RGV genome browser by choosing a track of interest. Users can also upload their own datasets (several file formats are allowed: gff3, gtf, bigWig, bam, and vcf) in the JBrowse session to compare them to the existing tracks by using the option “Open” in the “File” tab. If necessary the user can first run the RGV data processing pipeline, implemented in the Galaxy session and then upload their own tracks into the system.

RGV working environment (Galaxy)
details

To host genomics tools essential for data comparisons between genome releases and above all between species, we implemented a Galaxy bioinformatics workflow environment. Galaxy is an open web-based platform for genomic research that provides users with an easy-to-use web interface to create complex biological workflows by tools that simply need to be dragged and dropped. The RGV Galaxy session is available without creating an account

RGV data processing
+
Description

This pipeline is based on the implementation of three tool suites: UCSC tools, bedtools, and CrossMap. The former is used for all data file format conversions in either bedGraph or bigWig formats. The second is employed for the data standardization step. Finally, the latter is used in both cross-assembly and cross-species conversions of genome coordinates and makes use of pair-wise alignment files (chain format) provided by UCSC. The entire process takes roughly 30 min for an input file (bam format) of 200 Mb. Once the conversion is completed, the user can easily upload the resulting bigWig file to the RGV JBrowse session

RGV alignment tool
+
Description

The genome alignment workflow based on the Blast-Like Alignment Tool (BLAT) allows users to use a one-step procedure to automatically align their DNA/RNA or protein sequences (fasta format) onto the thirteen reference genome sequences available in the RGV system. The resulting alignments are post-processed and made available in two forms: a table including direct links to the JBrowse session and a General Feature Format (gff) file that can be uploaded to the genome browser.
